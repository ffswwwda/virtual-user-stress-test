# 运行原理（How It Works）· 详述

本文档是虚拟用户压力测试引擎的完整理论说明，对应网站「运行原理」页面。

## 一、问题：只有壳子，没有内核

最初版本把"用户原话"和"你的问题"直接拼在一起，没有中间推理层：

- 你输入「60美元飞机杯，主打静音」，返回的反对声音里却出现「震动很弱」（与飞机杯静音无关）、「60欧元花在妓女身上」（其他品类）——因为系统没有判断"这条评价跟当前场景是否相关"。
- 立场结论也是模板（`原话 → 对该场景偏反对，需关注其XX`），没有分析这个关键词是否真的跟场景有关。
- 没有驱动因子归因：好的压力测试应告诉你"这个用户为什么反对"。

## 二、完整内核：四层引擎（以有大模型为前提）

### Layer 1 · 场景分解（Scenario Decomposition）
**理论：联合分析（Conjoint Analysis）**——把复杂方案拆为独立可评维度。
LLM 把自由文本拆成：价格点 $60 / 核心卖点 静音 / 品类 飞机杯 / 场景类型 新品上架 / 渠道暗示 线上。
纯规则用正则 + 关键词词典即可提取 80%。

### Layer 2 · 用户画像深化（Persona Enrichment）
**理论：Kahneman 参考依赖 + 保护动机理论**。
从原始 VOC 预计算每个用户的结构化字段：参考价 refPrice、痛点词 barriers、需求标签 needs、强度分 intensity、竞品偏好、星级。
纯规则：价格正则提取、标签从现有字段直接取、强度是纯数学公式。

### Layer 3 · 核心匹配引擎（Attribute-Persona Matching）★
**理论：ELM 精细可能性模型**。
对每个用户 × 场景计算：
- **相关性评分**：场景属性集合 ∩ 用户标签集合。零相关 → 该评价不应出现在结果里。
- **价格敏感度**：有参考价则对比；无参考价则标记"价格未知"。
- **立场概率**：P(反对) = f(相关性, 星级, 价格差, 情绪)。
- **响应特异性分级**：L1 泛化 / L2 属性级 / L3 叙事级。

### Layer 4 · 响应生成（Response Generation）
**理论：叙事一致性 + 认知失调**。
输出：立场 + 置信度、驱动因子（具体哪个属性触发了反对）、模拟叙事（LLM）、风险标注。

## 三、纯规则简化路径

| 层 | 有 LLM | 纯规则近似 | 纯规则能做？ |
|----|--------|-----------|------------|
| L1 | LLM 理解场景 | 正则+关键词词典 | ✓ |
| L2 | LLM 深度画像推理 | 结构化字段+关键词索引 | ✓ |
| L3 | 语义匹配(embedding)、多属性效用、ELM路径 | 关键词重叠评分、价格差公式、加权公式 | ✓ |
| L4 | LLM 原创叙事 | 模板增强（注入最相关原话+场景） | △（半） |

**纯规则能做到**：场景关键词提取、用户关键词索引、相关性过滤、价格敏感度判定、基于公式的立场概率、驱动因子归因、风险排序、聚合洞察。
**纯规则做不到（需 LLM）**：语义等价推理（「静音」≈「吵」≈「声音大」——用同义词类别词典部分缓解）、原创模拟叙事、反事实推演、言外之意检测、群体动力学。

## 四、核心公式推导

```
相关性 relevance = |用户关注维度 ∩ 场景卖点维度| / |用户关注维度|

立场得分 oppose = 星级基线
  + Σ(cat in 卖点维度){
       用户[cat]==barrier ? −0.16   // 痛点被解决 → 支持
      :用户[cat]==need   ? −0.10   // 需求被满足 → 支持
      :                     +0.05   // 触及但未命中
    }
  + (价格偏差: 有参考价且 ratio>1.15 → +min(0.30, ratio−1); ratio<0.85 → −0.10)
  + Σ(用户有 barrier 维度 且 场景未触及 ? +0.05)   // 未解决痛点余量
  + (情绪: 负+0.10 / 正−0.05)

立场 = oppose≥0.52 → 反对 ; ≤0.40 → 支持 ; 否则 中立
置信度 = clamp(|oppose−0.46|×2.2, 0.4, 0.98)
风险 = (立场=反对) ? 影响力×(0.5+强度) : 0
```

**为什么"静音"卖点会让"噪音大"用户转支持**：该用户关注维度含 `quiet`（痛点 barrier），场景卖点维度也含 `quiet`，命中且是 barrier → oppose −0.16 → 立场向支持移动。反之"震动很弱"用户关注 `function`，场景卖点是 `quiet`，无交集 → 相关性极低，不会被误判为高度相关反对。

## 五、输出形态：双页结构

- **用户清单**：N 张独立身份卡（默认 100），锚定层（真实 VOC 原话+标签）、标签层（对齐 VOC 九大模块）、生成层（私态度+say-do gap+人口代理），压力测试后附立场徽章+驱动因子+相关度。支持分群/立场/极端筛选与搜索。
- **数据看板**：立场分布条、分群×立场矩阵、反对原因聚类 Top8、高风险用户排名（影响力×强度）。两页共享同一批运算结果，可切换对照。
