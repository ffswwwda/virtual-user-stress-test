# 虚拟用户压力测试 · Virtual User Stress Test

基于真实 VOC 源头 + 理论驱动匹配引擎的「上市前挑刺」工具。

## 是什么

在产品 / 内容真正推向市场之前，用一群"虚拟但真实溯源"的用户，对它做一轮压力测试——挑刺、找漏洞、发现极端风险。

## 在线体验

https://ffswwwda.github.io/virtual-user-stress-test/

## 核心特性

- **真实锚定**：每位虚拟用户的核心声音来自真实 VOC（本仓库内置飞机杯·淋浴支架 1731 条真实评价），可溯源、不捏造。
- **理论分布**：Rogers 创新扩散五档 + von Hippel 领先用户 + Pareto 影响力权重。
- **匹配引擎内核**：场景解析（价格/卖点/品类）→ 与用户真实标签做相关性匹配 → 算出立场、驱动因子、风险，而非随机拼贴。
- **双页输出**：用户清单（N 张独立身份卡，可筛选搜索）+ 数据看板（立场分布、分群×立场、反对原因聚类、高风险排名）。
- **运行原理页**：完整披露四层引擎架构与纯规则公式。

## 本地构建 VOC 数据集

```bash
python3 references/build_voc.py
# 输出 voc-source.js（window.DEMO_VOC），需提供带标签列的 CSV
```

标签列建议对齐 voc-report 九大模块：用户画像 / 购买动机 / 购买目的 / 使用场景 / 使用阻碍 / 克服方式 / 用户忠诚度 / 产品改进建议 / 十三种需求。

## 部署

纯静态站点（index.html + voc-source.js + favicon.svg）。推到 GitHub 后开启 Pages（main 分支）即可。Skill 说明见 `SKILL.md`，运行原理见 `references/how-it-works.md`。

## 能力边界

- ✅ 可测：概念测试、功能测试、文案/卖点测试、价格敏感度
- ❌ 不可虚拟化：真人体验（手感/流畅度/情绪流）
- 🔗 前置真实反馈输入 → 可测（虚拟人复用其洞察做放大推演）
