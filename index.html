这是什么

在把产品 / 内容真正推向市场之前，先用一群"虚拟但真实溯源"的用户，对它做一轮压力测试——挑刺、找漏洞、发现极端风险。

我们的逻辑

四层结构：真实锚定 → 理论分布 → 算法生成 → 压力测试
01

真实锚定

每一位虚拟用户的核心声音都来自真实 VOC 源头（用户评价 / 社媒反馈），可溯源、不捏造。

02

理论分布

群体结构遵循 Rogers 创新扩散、von Hippel 领先用户、Pareto 影响力等理论，不是均匀抽样。

03

算法生成

在真实声音之上，生成层补充私态度、人口代理等带置信度的推断字段，明确标注"哪些是生成的"。

04

压力测试

输入你的方案，匹配引擎逐用户计算相关性与立场，输出支持 / 中立 / 反对与驱动因子。

支撑理论

群体结构背后的四个理论支点

Rogers 创新扩散理论

用户接受新事物呈五档分布：创新者 2.5% / 早期采纳者 13.5% / 早期大众 34% / 晚期大众 34% / 落后者 16%。

von Hippel 领先用户

约 10% 极端 / 领先用户，其痛点与改进建议往往预示主流未来需求，是创新信号的集中承载者。

Pareto / Heavy-Half

影响力高度不均：少数用户（KOL、高赞者）贡献多数声量与传播力，故采用 Pareto 权重。

Ajzen + 社会期望

计划行为理论说明"说的"与"做的"存在 say-do gap；社媒表达还受社会期望偏差影响，故生成层标注置信度。

适用场景

什么时候该做一轮压力测试

新品上市前试用

把概念丢给这群用户，看哪些人会挑刺、哪些痛点没被解决。

社媒内容挑刺

一篇种草 / 测评文案，哪些用户会反感、会反驳、会传播？

极端风险发现

识别高影响力 + 高负面情绪的用户，提前预判舆情雷点。

能力边界（很重要）

什么能测、什么不能测，必须说清楚

可直接压力测试

概念测试、功能测试、文案 / 卖点测试、价格敏感度——只要能从 VOC 提取标签，就能算。

真人体验 · 不可虚拟化

手感、流畅度、情绪流、真实使用体验——这些是真人的，虚拟用户无法替代。

前置真实反馈 → 可测

若你已收集真人的体验反馈描述，作为输入喂给系统，虚拟用户可复用其洞察做放大推演。

运行原理

虚拟用户压力测试不是随机拼贴评价，而是一套有理论、有运算逻辑的系统。下面把"从 VOC 到压力测试结论"的完整链路说清楚——先讲有大模型时的完整内核，再讲纯规则下能做到什么程度，最后给出输出形态。

问题：只有壳子，没有内核

最初的实现把"用户原话"和"你的问题"直接拼在一起，没有中间推理层：你输入「60美元飞机杯，主打静音」，返回的反对声音里却出现「震动很弱」（与飞机杯静音无关）、「60欧元花在妓女身上」（其他品类）——因为系统没有判断"这条评价跟当前场景是否相关"。立场结论也是模板，不是推理。

完整内核：四层引擎（以有大模型为前提）

Layer 1 场景分解：把你的自由文本拆成结构化属性（价格 / 卖点 / 品类 / 场景类型），理论依据是联合分析（Conjoint）——复杂方案可拆为独立可评维度。
Layer 2 用户画像深化：从原始 VOC 预计算参考价、痛点词、需求标签、强度分，理论依据是 Kahneman 参考依赖 + 保护动机理论。
Layer 3 核心匹配引擎 ★：对每个用户 × 场景计算相关性评分、价格敏感度、立场概率、响应特异性级别，理论依据是 ELM 精细可能性模型。
Layer 4 响应生成：以该用户口吻生成回应，含驱动因子归因与风险标注，理论依据是叙事一致性 + 认知失调。

纯规则下能做到什么程度

当前网站走的是纯规则路线（无大模型调用）。以下映射说明每一层如何用规则近似：

纯规则能做到：场景关键词提取、用户关键词索引、相关性过滤、价格敏感度判定、基于公式的立场概率、驱动因子归因、风险排序、聚合洞察。
纯规则做不到（需 LLM）：语义等价推理（「静音」≈「吵」≈「声音大」——我们用同义词类别词典部分缓解）、原创模拟叙事、反事实推演、言外之意检测、群体动力学。

匹配引擎的核心公式（纯规则实现）

对每个虚拟用户，先用标签体系将其痛点 / 需求映射到关注维度（价格 / 静音 / 材质 / 耐用 / 清洗 / 尺寸 / 外观 / 功能 / 售后 / 舒适）；再解析场景卖点映射到同一维度集合。两者做交集即"相关性"。

相关性 relevance = |用户关注维度 ∩ 场景卖点维度| / |用户关注维度|
立场得分 oppose = 星级基线 + Σ(卖点契合其痛点 ? −0.16 : 卖点契合其需求 ? −0.10 : 未触及 ? +0.05) + 价格偏差 + 未解决痛点余量 + 情绪基调
立场 = oppose≥0.52→反对 ; ≤0.40→支持 ; 否则中立
置信度 = clamp(|oppose−0.46|×2.2, 0.4, 0.98)
风险 = (立场=反对) ? 影响力×(0.5+强度) : 0

关键点：若用户痛点是"噪音大"，而场景卖点是"静音"——维度命中且是痛点被解决，该用户反而转为支持，而不是被错判为反对。这正是此前空壳实现缺失的逻辑。

输出形态：双页结构

工作台生成后，结果组织为两个可切换的页面：
用户清单——真的有 N 张独立身份卡（默认 100），每张可溯源到真实 VOC 原话，支持按分群 / 立场 / 极端筛选与搜索；压力测试运行后，每张卡显示其立场徽章、驱动因子与相关度。
数据看板——聚合视图：立场分布、分群×立场矩阵、反对原因聚类、高风险用户排名。两个页面共享同一批运算结果，可来回切换对照。
