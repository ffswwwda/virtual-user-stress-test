THEORY · 理论支撑

用户群的形态，不是拍脑袋

「100 人里 10% 极端、50% 无所谓」背后，是四套成熟的行为科学理论。它们共同决定了生成的虚拟用户群如何分布、谁该被重点对待。

Rogers 创新扩散理论

采纳者天然呈钟形偏态五档：创新者 2.5% / 早期采纳 13.5% / 早期大众 34% / 晚期大众 34% / 落后者 16%。它决定了「这 100 人里，多少人会动」。
用在哪：默认群体分布形态。

von Hippel 领先用户理论

极端 / 领先用户当下的需求，往往是主流市场未来的需求。所以约 10% 的极端用户不是噪声，而是信号——他们定义产品下一步。
用在哪：极端用户标记与需求前瞻。

Pareto / Heavy-Half

哪怕落后者占 16%，真正驱动传播的是前 15%+ 的领先 / 早期采纳者。因此每个用户携带「影响力权重」，不是均匀 1。
用在哪：仿真时的带动力与聚合显著性。

Ajzen 计划行为 + 社会期望

人「说的」和「私下想的 / 真正做的」常有 gap（say-do gap）。所以卡片分「公开声音（真实）」与「私态度（生成）」，私态度才是动作预测的依据。
用在哪：生成层的私态度与动作倾向。

USE CASES · 使用场景

什么时候该用压力测试，而不是预测

凡是「上市 / 投放前，想先知道哪里会翻车」的时刻，都适合先跑一轮虚拟用户压力测试。

新品上市前试用

把产品卖点丢给这群虚拟用户，看极端用户会不会挑刺、价格敏感者会不会嫌贵、沉默大多数会不会无感。

社媒内容挑刺

一条内容发出去，谁会转、谁会喷、谁划走？在真实投放前用虚拟用户群预演舆论反应。

极端风险发现

专门放大领先用户与差评传播者，提前暴露可能被放大的负面叙事与合规 / 舆情隐患。

VS · 与预测世界的区分

压力测试 vs 预测世界，不是同一件事

你的理解基本正确——一个在上市前、一个在上市后；一个看计划的脆弱点、一个看市场的演化结果。两者互补，不替代。

上市前 · 试用与挑刺

虚拟用户压力测试

  • 找一批虚拟用户来试用、评价、挑刺
  • 发现极端反应、被拒风险、沉默大多数
  • 目的是「计划的健壮性」——哪里会破
  • 社媒同样适用：内容发出去前先预演舆论
VS
上市后 · 演化与预测

预测世界（用户研究预测引擎）

  • 把这群用户放进市场，看他们怎么演化
  • 预测传播 / 采纳的「结果量级」
  • 目的是「效果的预测」——会到哪
  • 多智能体仿真 + 公式模型双引擎

一句话：压力测试 = 找用户来试（看漏洞），预测世界 = 看市场怎么跑（看结果）。先用压力测试排雷，再用预测世界估量，是更稳的研发节奏。

压力测试引擎的四层内核与两种实现路径

PRINCIPLE · 运行原理

压力测试引擎是怎么算的

本工具不是「随机抽几条评价拼上去」，而是有四层可解释内核：场景被拆成属性，用户从 VOC 锚定真实原话，再用理论权重做匹配与立场推断，最后给出带归因的结果。下面逐层说明（当前版本为纯规则实现，LLM 增强作为可叠加的后续层）。

01

场景分解

把你的自由描述拆成结构化属性：价格点、核心卖点、品类、场景类型、渠道。纯规则即可完成 80%（正则 + 词典）。

02

用户画像深化

从原始 VOC 预计算结构化字段：参考价、痛点词、需求标签、强度分、竞品偏向。锚定层始终真实。

03

匹配引擎 ★内核

计算每个用户 × 场景的「相关度 + 价格敏感度 + 立场概率」，并归因驱动因子。这是整个系统的核心。

04

响应生成

输出立场、置信度、驱动因子、风险标注、相关度。纯规则给结构；LLM 可叠加原创叙事。

CORE · 核心匹配逻辑

一条虚拟用户，如何对场景表态

对每个用户，引擎做三件事：先看「这条评价跟你说的场景相不相关」，再看「价格差多少」，最后加权算出立场概率并归因。不相关的评价会被降权，不会硬塞进结果。

相关度评分

把场景关键词（如「静音 / 噪音 / 安静」）与用户的痛点词、需求标签、原话做集合交集；命中越多越相关。完全不命中的评价 → 相关度近 0，不参与反对判定。

价格敏感度

从原话提取参考价（如「才 30 不到」→30）。场景价与参考价的偏差越大，价格反对越强；未提参考价则标记为「价格未知」。

立场概率（加权公式）

P(反对) = w₁·星级反向 + w₂·相关度×负面 + w₃·价格偏差 + w₄·情绪标签。置信度即概率值；再按「影响力×强度」排风险，高风险优先展示。

当前不接大模型也能做的，以及必须 LLM 才能做的——诚实列出，决定了结果的可信度。

BOUNDARY · 能力边界

什么能测，什么不能，什么前置后可测

工具诚实地划清边界：态度与需求可压力测试；真人体验不可虚拟化；但真实体验反馈可作为锚定输入被复用。这也是虚拟用户卡「锚定层真实 / 生成层带置信度」设计哲学的直接体现。

可直接压力测试

新品概念测试、功能价值测试、文案 / 卖点反应。虚拟用户基于 VOC 锚定的分群、立场、需求生成反应，确定性可复算。

真人体验 · 不可虚拟化

使用手感、流畅度、情绪流、生理 / 直觉反应。必须真人真实场景，数据无法生成体感——这是工具的天花板。

前置真实反馈 → 可测

把真人体验反馈描述作为输入（锚定层），虚拟用户群复用它：测普遍性、分群差异、严重度。真实反馈是锚，虚拟人是放大器。

桥接点：体验不能「由虚拟人体验」，但体验反馈能「被虚拟人复用」。前置输入后，生成时可注入锚定层，让虚拟用户群放大已知体验问题的分布与影响。
